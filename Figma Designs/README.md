# Figma UI/UX Design – Stock Overflow

## Overview

This folder contains the **Figma UI/UX designs for the Stock Overflow Inventory Management System**.  
The designs were created using **Figma** to plan the user interface, system navigation, and user interaction flow before implementing the frontend of the application.

The design includes **high-fidelity UI screens** that represent the major modules of the system such as **authentication, inventory management, supplier management, POS operations, and administrative controls**.

These designs provide a clear visual representation of the platform and help ensure that the development phase follows a **structured and consistent UI design**.

---

## UI Design

The UI design was created as **high-fidelity designs directly in Figma**, incorporating the system’s visual theme, layout structure, and reusable UI components.

### Design Focus

The design focuses on:

- Clear and intuitive user interface
- Consistent color scheme and design system
- Structured dashboard layouts
- Efficient data tables and inventory management interfaces
- Clear navigation between system modules

---

## Design System

The UI design follows a **consistent color palette suitable for enterprise dashboard applications**.

### Primary Colors

| Element | Color |
|------|------|
| Primary Color | `#2563EB` |
| Sidebar Color | `#1E3A8A` |
| Background Color | `#F9FAFB` |
| Card Background | `#FFFFFF` |

### Text Colors

| Type | Color |
|------|------|
| Primary Text | `#111827` |
| Secondary Text | `#6B7280` |

### Border Color

`#E5E7EB`

### System Status Colors

| Status | Color |
|------|------|
| Success (In Stock) | `#22C55E` |
| Warning (Low Stock) | `#F59E0B` |
| Danger (Out of Stock) | `#EF4444` |

These colors are used consistently throughout the application to visually represent **system status and inventory conditions**.

## Purpose

The Figma designs act as a **visual blueprint for the development of the Stock Overflow platform**.

They help developers understand:

- Page layouts
- UI components
- Navigation structure
- User interactions
- Data presentation for inventory management

By designing the interface before development, the system maintains **design consistency, usability, and clarity throughout the implementation phase**.
